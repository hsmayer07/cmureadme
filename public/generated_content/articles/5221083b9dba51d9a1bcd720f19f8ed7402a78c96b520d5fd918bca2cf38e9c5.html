Task 2

  1. `default_nettype none
  1. This ensures no ambiguous wires (wires that don’t have an assigned type like logic).
  1. Module instantiation creates a copy of the hardware with the appropriate inputs/outputs that will continuously run as the program specifies. A function call, however, only runs a single time for every call made.
  2. The logic keyword ensures that the signals handled by the program are digital signals (outputs are either 0 or 1 under typical operating conditions). This is a requirement for the logic gates that are used to build the systems.
  3. Only a single instance is allowed to run. It’s unpredictable which instance will run if the simulation is run many times.
  4. A top module handles the execution of the created modules (can instantiate devices under test as well as a testbench). It has no inputs or outputs because there is no reasonable system to handle them. Instead, the testbench must generate the input combinations and measure the outputs to ensure that the project works as intended.
  5. Both modules are equivalent because the hardware runs in parallel. The simulation uses scheduling to ensure that this is maintained when simulating.
  6. The waveform viewer is useful because it shows how various logic signals inside of the program change over time/react to inputs. It is important when debugging errors in code.
  7. The logic signals are instantiated by the system and then modified by the muxTester module as a way to test the simulation
  8. The inputs to the muxTester are the outputs of the mux based on the provided inputs for the test.

Task 3

  1. Synthesis generates specific code that is used to program a specific FPGA (specifies the gates, pins, and connections). Simulation virtually runs the project to measure its performance, test, and debug the code.
  2. Providing a pin assignment provides the synthesis tool with meaning for the various input and output connections that are specified. This ensures that the tool targets the proper board and uses the appropriate resources when the synthesized code is running on-chip.
  3. The pin planner will be colored in and provide the location of each of the pins whose configuration has been imported. When attempted, the code was synthesized and downloaded to the board but it was obvious that there was an issue because the input/output of the board was not responsive as designed/tested.
  4. The project cannot be synthesized and Quartus will return an error message instead.
  5. When attempted, the code compiled and was downloaded to the board without issue, but there was no way for the testbench to run on the hardware. However, this seems unusual as an error should have been generated instead of compiling.
